## 1. Curie: Toward Rigorous and Automated Scientific Experimentation with AI Agents

- **Authors & Venue**: Patrick Tser Jern Kon, Jiachen Liu, Qiuyi Ding, Yiming Qiu, Zhenning Yang, Yibo Huang, Jayanth Srinivasa, Myungjin Lee, Mosharaf Chowdhury, Ang Chen — **NeurIPS 2025 Spotlight**
- **arXiv**: 2502.16069

### Core Problem

现有 LLM agent 在自动化科学研究中已经展现了一定能力（如文献搜索、代码生成），但**严格的实验执行**（rigorous experimentation）这一步几乎没有被认真对待。问题不是"能不能跑代码"，而是"能不能像一个合格的研究者一样设计对照实验、控制变量、验证结果的可靠性"。这个问题重要是因为实验的严谨性直接决定了科学发现的可信度——如果 AI agent 跑出来的实验结果不可复现或者实验设计有缺陷，那自动化科研就是空中楼阁。

### Key Contribution

Curie 提出了一个**将实验严谨性嵌入 agent 架构**的框架，而不是把严谨性作为事后检查。核心创新是三层 rigor 机制：(1) Intra-agent rigor 确保单个 agent 的每一步输出都经过验证（实验设置验证器 + 执行验证器）；(2) Inter-agent rigor 通过细粒度的计划分割和控制流强制，确保多 agent 协作时实验逻辑不被破坏；(3) Experiment Knowledge Module 维护结构化的实验记录，像"时光机"一样追踪假设的演变过程。这不是简单的 multi-agent orchestration——它把"什么是严格的实验"这个科学方法论问题形式化成了 agent 系统的架构约束。

### Method

系统由一个 Architect Agent（负责高层规划）和多个 Technician Agent（负责具体实现）组成。Architect 接收实验问题后，先生成实验计划，然后 Inter-Agent Rigor Module 将计划分割成独立的变量子集（fine-grained plan partitioning），每个子集分配给一个 Technician 并行执行。关键设计选择是**控制流强制**（Control Flow Enforcement）：系统不允许 agent 跳过实验步骤或者改变变量控制关系，这直接防止了"agent 为了省事跳过对照组"这种常见失败模式。

Experiment Knowledge Module 用分层写入权限（Tiered Write Access）限制每个 agent 只能修改自己角色范围内的实验记录——Technician 不能改 Architect 的假设，Architect 不能直接改实验数据。这个设计借鉴了真实实验室的权限管理，用系统架构保证实验记录的完整性。

### Results

在 46 个跨 4 个 CS 领域的实验问题上，Curie 的**实验结论正确率为 36.1%**，比最强 baseline OpenHands (10.5%) 高 3.4 倍。更值得注意的是分阶段对比：实验设计阶段 Curie 达到 97.9%（几乎完美），但从设计到结论的 pipeline 逐步下降（执行 78.1% → 对齐 73.4% → 结论 36.1%），说明瓶颈在执行和推理而不是规划。Magentic-One 的结论正确率仅 2.3%，说明没有 rigor 机制的 agent 几乎不可能做出可靠的实验结论。

### Multi-Perspective Review

**Domain Expert**: 这篇论文的 novelty 在于把"实验严谨性"这个科学哲学概念工程化为 agent 架构约束。之前的 AI4Science 工作要么关注 hypothesis generation（AI Scientist），要么关注 code generation，很少有人认真思考实验执行的严谨性。获得 NeurIPS Spotlight 说明了这个问题的重要性被认可。但 benchmark 规模偏小（46 个问题），且仅限 CS 领域。

**Methodology Critic**: 36.1% 的绝对准确率仍然很低——如果两次实验中有将近两次是错的，这在真实科研中是不可接受的。另外评估用了 LLM judge，虽然和人工标注做了对比，但 LLM judge 本身的 reliability 在复杂实验场景下存疑。最关键的问题是：benchmark 的 ground truth 来自已发表论文，但科学实验的正确答案经常不是唯一的。

**Visionary**: 这篇工作的长远意义在于定义了"rigorous experimentation"这个评估维度。未来的 AI research agent 不仅要能跑实验，还要能证明自己的实验是严格的。这可能会催生一个新的 sub-field：Automated Experimental Methodology。

### Build-On Directions

1. **跨学科扩展**：把 rigor 框架扩展到物理/化学/生物实验，需要处理实验时间尺度、安全约束、不确定性量化等 CS 实验没有的维度
2. **Self-improving rigor**：让 agent 从失败的实验中学习什么样的 rigor check 是有用的，自适应调整验证策略
3. **Rigor as reward**：把实验严谨性作为 RL reward signal，训练 agent 内化严谨性而不是依赖外部 validator

### Trend Signal

反映了 AI4Science 从"能不能做"到"做得好不好"的转变。2024 年的 AI Scientist 证明了"AI 能做实验"，2025 年的 Curie 开始回答"AI 能做严格的实验吗"。下一步是"AI 能做创新的实验吗"。

## 2. EXP-Bench: Can AI Conduct AI Research Experiments?

- **Authors & Venue**: Patrick Tser Jern Kon, Jiachen Liu, Xinyi Zhu, Qiuyi Ding, Jingjia Peng, Jiarong Xing, Yibo Huang, Yiming Qiu, Jayanth Srinivasa, Myungjin Lee, Mosharaf Chowdhury, Matei Zaharia, Ang Chen — **ICLR 2026**
- **arXiv**: 2505.24785

### Core Problem

自动化 AI 研究的潜力巨大，但我们**没有一个严格的 benchmark 来衡量 AI agent 能不能真正做完一个完整的 AI 研究实验**。现有 benchmark 要么只测代码生成（SWE-Bench），要么只测单步推理，没有人测"从研究问题到实验结论"的全流程。这个问题重要是因为：如果连测都测不了，就无法系统性地改进。

### Key Contribution

EXP-Bench 是第一个从真实 AI 论文中提取完整实验 pipeline 的 benchmark。它不是让 agent 解一个编程题，而是给一个研究问题 + 不完整的 starter code，要求 agent 完成**假设制定 → 实验设计 → 代码实现 → 执行 → 结果分析 → 得出结论**的全流程。从 NeurIPS 和 ICLR 2024 的 51 篇论文中提取了 461 个实验任务，包含 12,737 个可独立评分的子任务。关键创新是 semi-autonomous extraction pipeline：用 LLM 从论文 + 代码中自动提取实验规格，但有人工验证环节。

### Method

任务提取分两阶段：(1) Multi-modal extraction 从论文（文本 + 图表）中提取核心实验要素（研究问题、期望结果、数据集、评估指标）；(2) Implementation extraction 从 GitHub repo 中定位相关代码并组装成 starter code + ground truth solution（通过 git diff 表示代码修改）。每个任务的 ground truth 由三部分组成：实验设计规格、代码修改、实验结论。评估 pipeline 独立对三个阶段打分：设计是否正确、代码修改是否正确、结论是否正确。

设计上的一个重要选择是**给 starter code 而不是从零开始**——这模拟了真实的研究场景（你总是在已有代码基础上做实验），同时降低了纯工程难度，让评估聚焦在实验能力而不是编程能力。

### Results

结果令人清醒：最强的 LLM agent（OpenHands, IterativeAgent）在单独的实验维度上偶尔能达到 20-35%，但**完整可执行实验的成功率只有 0.5%**。这意味着在 461 个实验任务中，AI agent 只能完整做对 2-3 个。这不是 agent 不够聪明，而是端到端实验要求的能力组合太复杂：你需要同时会设计、会实现、会调试、会分析——任何一环出错，整个实验就失败了。

### Multi-Perspective Review

**Domain Expert**: 这是 Curie 团队（同一批作者）的 companion paper。Curie 提出方法，EXP-Bench 提出评估。两篇一起读能看到完整的图景。461 个任务的规模足够大，覆盖了 RL、生成模型、AI 应用等多个子领域。ICLR 2026 接收说明 community 认为这个 benchmark 是有价值的。

**Methodology Critic**: 0.5% 的成功率让人怀疑 benchmark 是否过难了。如果所有方法都接近零分，那这个 benchmark 区分不了方法的好坏。另外，ground truth 的定义依赖于原论文的实验设计——但原论文的实验设计不一定是唯一正确的方式。一个 agent 用不同但同样合理的实验设计，会被判为错误。

**Visionary**: EXP-Bench 的真正价值可能不在于排行榜，而在于**诊断**：通过 12,737 个子任务的细粒度评分，我们可以精确知道 AI agent 在实验 pipeline 的哪个环节最弱。这为有针对性地改进提供了 roadmap。

### Build-On Directions

1. **Partial credit evaluation**：不是 0/1 判定，而是给出实验完成的程度分（比如"设计对了 80%，实现对了 50%，结论方向对但数值错"）
2. **Multi-agent collaborative benchmark**：测试多个 specialized agent（设计 agent + 代码 agent + 分析 agent）协作的表现
3. **Longitudinal tracking**：用 EXP-Bench 追踪不同 LLM 版本在实验能力上的进步曲线

### Trend Signal

AI research automation 从"demo 阶段"进入"measurement 阶段"。有了 benchmark，才能有 leaderboard，才能有系统性改进。这和 ImageNet 之于 CV、SWE-Bench 之于代码生成的作用类似。

## 3. Sci-Reasoning: A Dataset Decoding AI Innovation Patterns

- **Authors & Venue**: Jiachen Liu, Maestro Harmon, Zechen Zhang (Orchestra Research) — Submitted to **ACL**, arXiv 2601.04577
- **Date**: January 2026

### Core Problem

AI 创新在加速，但**创新背后的思维过程**（研究者如何发现 gap、综合前人工作、产生新 insight）仍然是黑箱。我们有大量论文数据，但没有结构化的数据来描述"从 A 论文到 B 论文，研究者做了什么样的思维跳跃"。这个数据缺失直接阻碍了两件事：(1) 对科学进步的定量研究；(2) 训练能做科学推理的 AI agent。

### Key Contribution

Sci-Reasoning 是第一个捕捉高质量 AI 研究背后**思维综合过程**的结构化数据集。它不只是 citation graph（谁引用了谁），而是 reasoning trajectory（从前驱论文到当前论文，研究者做了什么样的推理）。从 NeurIPS、ICML、ICLR 2023-2025 的 Oral 和 Spotlight 论文中追溯知识谱系，标注每篇前驱论文的角色（Baseline、Inspiration、Gap Identification、Foundation）和关系类型。

最重要的发现是**15 种创新思维模式**，其中三种占了 52.7%：Gap-Driven Reframing（24.2%）、Cross-Domain Synthesis（18.0%）、Representation Shift（10.5%）。这意味着近四分之一的顶会 oral/spotlight 论文的核心创新是"重新框定问题"。

### Method

四阶段 LLM-accelerated + human-verified pipeline：(1) 用 community-validated quality signal（Oral/Spotlight 标记）筛选高质量论文；(2) 用 LLM 追溯每篇论文的关键前驱；(3) 生成结构化推理轨迹，标注前驱的角色和关系；(4) 人工验证。

在 model ablation 中比较了 GPT-5.2、GPT-5、GPT-5-mini、GPT-4.1，发现 GPT-5 在成本-质量上最优（89.73% recall）。整个数据集 3,819 篇论文（999 Oral + 2,820 Spotlight），发布在 HuggingFace 上。

值得注意的是构建效率：作者提到 10 天内完成，使用了 fully transparent AI Scientist workflow，说明这种"LLM-accelerated 研究"本身正在成为一种可行的研究范式。

### Results

15 种创新模式中，最常见的三种组合（"创新配方"）：Gap-Driven Reframing + Representation Shift、Cross-Domain Synthesis + Representation Shift、Gap-Driven Reframing + Cross-Domain Synthesis。这些组合的共同特点是**先改变看问题的方式，再引入技术方案**——不是直接改方法，而是先 reframe 问题。

### Multi-Perspective Review

**Domain Expert**: 这篇论文的独特价值在于把 Science of Science 带入了 AI 领域。之前有 citation network analysis（关注结构），有 research trend analysis（关注主题），但很少有人关注"思维过程的结构化表示"。15 种创新模式的分类本身就是一个 contribution。不过分类的 validity 依赖于 LLM 的 annotation quality。

**Methodology Critic**: "Reasoning trajectory" 本质上是 LLM 对论文的解读，不是研究者真实的思维过程。研究者可能根本不是按 LLM 归纳的路径思考的——这是一种 post-hoc rationalization。另外 89.73% recall 的 predecessor extraction 意味着约 10% 的关键前驱被遗漏，对于 reasoning trajectory 的完整性是个问题。

**Visionary**: 如果 Sci-Reasoning 的 pattern 是准确的，那它直接可以用来构建"创新推理引擎"——给一个领域的 gap，系统性地尝试 15 种创新模式，生成候选研究方向。这可能比 brainstorming 更系统化。

### Build-On Directions

1. **Innovation pattern predictor**：给定一篇新论文的 related work，预测它最可能用了哪种创新模式
2. **Automated research ideation**：用 15 种 pattern 作为 template，给定一个 gap 自动生成候选 research ideas
3. **Longitudinal analysis**：追踪不同创新模式在不同时期的流行度变化，看 AI 领域的创新方式是否在演变

### Trend Signal

Meta-research（研究"研究"本身）正在被 LLM 加速。10 天造一个 3,819 篇论文的数据集在以前不可想象。同时反映了"LLM as research tool"这个趋势——不只是辅助写代码，而是辅助理解科学创新的结构。

## 4. Towards General Agentic Intelligence via Environment Scaling (AgentScaler)

- **Authors & Venue**: arXiv 2509.13311, 与 NeurIPS 相关
- **Note**: Paper 标题 "Environments" 最可能对应的是 environment scaling 方向的工作

### Core Problem

当前 LLM agent 的能力提升主要靠三条路：更大的模型、更多的数据、更好的算法。但有一个维度被严重忽视：**环境的规模和多样性**。Agent 是在环境中学习的，如果训练环境单一（只有一个 benchmark），agent 的泛化能力就有上限。这个观点认为环境 scaling 是达到 general agentic intelligence 的关键瓶颈。

### Key Contribution

这篇工作提出 environment scaling 作为 agent 能力提升的第四条路线（parallel to model scaling, data scaling, algorithm improvement）。核心论点是：就像 LLM 需要在 diverse text 上预训练才能泛化，agent 需要在 diverse environments 上训练才能泛化。这不是简单的"多跑几个 benchmark"，而是需要 procedural generation、environment composition、difficulty curriculum 等系统化方法来生成足够多样的训练环境。

### Method

虽然具体方法因版本不同可能有差异，但 environment scaling 的核心框架是 **Generation-Execution-Feedback** 循环：环境生成任务 → agent 执行 → 环境反馈 → 用反馈改进 agent 和环境。相关的具体实现包括：OpenEnv（HuggingFace/Meta 的环境框架）、ARE（Scaling Up Agent Environments）、Terminal-Bench 2.0 等。

关键设计选择是 **RL over static datasets**：作者们认为 agent 应该从环境交互中通过 RL 学习，而不是从静态的 (observation, action) pair 数据集中学。这和 LLM 领域从 SFT 到 RLHF 的转变类似。

### Results

Snorkel AI 在 2026 年初的博客中将 environment scaling 称为"2026 年的主题"。具体数据取决于具体实现，但共识是：在更多样的环境上训练的 agent，在 held-out 环境上的 zero-shot 表现显著更好。

### Multi-Perspective Review

**Domain Expert**: Environment scaling 作为一个概念并不新（curriculum learning、procedural generation 都是老话题），但把它提升到和 model/data scaling 同等重要的位置是有 insight 的。Gaia2（Feb 2026）也在往这个方向走。

**Methodology Critic**: "More environments = better agents" 这个结论太模糊了。关键问题是：什么样的环境多样性是有用的？随机生成的环境可能大部分是 trivial 或 degenerate 的。需要更精确的理论来指导环境设计。

**Visionary**: 如果 environment scaling 成立，那"环境设计"会成为一个独立的研究方向——不是设计单个环境，而是设计"环境分布"。这可能会催生 World Model + Environment Generator 的组合。

### Build-On Directions

1. **Quality-aware environment generation**：不只是 scale up 数量，而是生成高质量、有教育意义的环境（类似 curriculum learning）
2. **World model as environment simulator**：用 world model 生成训练环境，形成 world model ↔ agent 的共进化
3. **Transfer metric**：定义环境之间的"距离"，预测在 environment A 上训练的 agent 能不能 transfer 到 environment B

### Trend Signal

从 model scaling 到 data scaling 到 environment scaling——AI 能力提升的瓶颈在不断转移。这暗示了一个更大的 pattern：当模型够大、数据够多之后，**learning context**（在什么环境中学、以什么方式学）成为新的瓶颈。

## 5. SentiAvatar: Towards Expressive and Interactive Digital Humans

- **Authors & Venue**: Chuhao Jin, Rui Zhang, Qingzhe Gao, Haoyu Shi, Dayu Wu, Yichen Jiang, Yihan Wu, Ruihua Song (Renmin University of China, SentiPulse) — arXiv 2604.02908, April 2026

### Core Problem

现有的 digital human / avatar 系统要么只能做简单的 lip sync（嘴对上了但身体不动），要么能做全身动作但和语音内容脱节。根本问题是**语义（说什么）和韵律（怎么说）对身体动作的影响是不同层级的**：语义决定"做什么动作"（说"过来"时招手），韵律决定"动作的节奏"（重音处动作幅度大）。现有方法把这两个层级混在一起，导致生成的动作要么语义不对要么节奏不对。

### Key Contribution

SentiAvatar 提出了一个**两阶段分层架构**（Plan-then-Infill），将语义对齐和韵律对齐解耦：Stage 1 用 LLM-based Motion Planner 从语义层面生成关键帧（keyframe），Stage 2 用 Audio-aware Infill Transformer 在关键帧之间插值，用语音的 frame-level 特征驱动细粒度动作。另一个重要贡献是 SuSuInterActs 数据集：21,133 clips, 36.9 小时的对话数据，同步了语音、全身动作（393 维）和面部表情（51 维 ARKit blendshape）。

Motion Foundation Model 在 200K+ 动作序列（676.4 小时）上预训练，backbone 是 Qwen-0.5B 加了 motion/audio token 词表。这个选择很聪明——用 LLM 做 motion 的"语言模型"，让 motion planning 可以利用 LLM 的 sequence modeling 能力。

### Method

Stage 1 (LLM Motion Planner)：输入是动作标签 + 稀疏音频 token，输出是 keyframe motion token（每 4 帧一个关键帧）。用 Residual VQ-VAE（4 层残差量化，codebook 512）把连续动作编码为离散 token。Backbone 是预训练的 Qwen-0.5B，扩展了 motion 和 audio 的 vocabulary。支持多轮对话的"continuation mode"。

Stage 2 (Audio-aware Infill Transformer)：在关键帧之间插值中间帧。条件是 dense HuBERT audio features（768 维，frame-level）。8 层 Transformer encoder，16 heads，512 维，约 38.5M 参数。训练时用 masked token prediction，推理时 6 步迭代 refinement。

面部表情是并行 pipeline：Face Infill Transformer 直接从音频生成面部 token。

推理延迟约 0.3 秒生成 6 秒输出，支持实时流式多轮对话。

### Results

在自建的 SuSuInterActs 数据集上，R@1 达 43.64%（vs MoMask 34.55%，+26.3%），FID 从 36.25 降到 8.912。用户研究（1-5 Likert）总体评分 2.99 vs MoMask 2.67（+12%），韵律同步评分提升最大（+14.5%）。

Ablation 显示 LLM Planner 和 Infill Transformer 的贡献最大（去掉任一个 R@1 从 43.64% 降到 ~28%），验证了分层设计的必要性。关键帧间隔 t=4 是最优的——t=2 给 LLM 太大压力，t=8 丢失语义对齐。

跨数据集泛化（BEATv2）FGD 4.941，说明模型不只是记住了 SuSuInterActs 的分布。

### Multi-Perspective Review

**Domain Expert**: 这篇论文的核心 insight——语义和韵律的解耦——在语音合成领域（FastSpeech 等）早已被验证，但在全身动作生成领域是第一次系统性地做。Motion Foundation Model 的规模（200K+序列）和预训练策略也是一个贡献。SuSuInterActs 数据集对中文 conversational gesture 研究有独特价值。

**Methodology Critic**: 用户研究的绝对分数偏低（最高 3.16/5），说明 digital human 的表现力离人类水平还有很大差距。另外数据集是用专业演员录制的，但真实对话中人的动作比演员"表演"的更subtle、更不规则，泛化到真实场景可能有 domain gap。
