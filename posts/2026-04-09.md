## 1. WAC: World-Model-Augmented Web Agents with Action Correction

- **Authors & Venue**: Zhouzhou Shen, Xueyu Hu, Xiyun Li, Tianqing Fang, Juncheng Li, Shengyu Zhang — arXiv 2602.15384
- **Links**: [arXiv](https://arxiv.org/abs/2602.15384) | [PDF](https://arxiv.org/pdf/2602.15384)

### Core Problem

现有的 web agent（基于 LLM 做网页操作的 agent）在做决策时有两个根本问题：(1) **认知孤立**——action model 不知道自己的操作会导致什么后果，也无法预测环境变化；(2) **风险盲区**——agent 可能贸然执行不可逆的高风险操作（比如提交表单、删除数据），导致任务失败且无法回头。这两个问题的根源是 web agent 缺乏一个"内在的环境模型"——它不能在脑子里预演操作后果。

### Key Contribution

WAC 提出了一个三模型协作框架：action model 负责生成候选操作，world model 作为"web 环境专家"提供策略建议并模拟操作后果，judge model 审查模拟结果并在必要时触发纠错反馈。核心创新是**两阶段推演链**（two-stage deduction chain）：先用 world model 做 consequence simulation（"如果我点这个按钮，页面会变成什么样？"），再用 judge model 评估模拟结果的风险等级（"这个后果可接受吗？"）。只有通过审查的操作才会被执行。这不是简单的 self-reflection——它用了专门的 world model 来预测环境变化，这比让同一个 LLM 既做决策又做反思要可靠得多。

### Method

系统分三层运作。第一层是**多 agent 协作**：action model 接收当前网页状态和任务描述，在生成操作之前先向 world model 咨询策略建议。World model 基于对 web 环境状态转移动态的先验知识，给出高层指导（"在这个页面上，你应该先填写表单再提交，而不是直接点提交"）。Action model 再把这些建议落地为具体的可执行操作。

第二层是**风险感知执行**。对于 action model 生成的每个候选操作，world model 模拟执行后的环境状态（"点击'确认'后，页面会跳转到支付页面，之前填的信息会被提交"）。Judge model 然后审查这个模拟结果，判断是否存在不可逆风险或偏离任务目标的情况。如果判定有风险，judge 会生成具体的纠错反馈，action model 据此修正操作。

这个设计的关键选择是**角色分离**：world model 专注于环境建模而不做决策，judge model 专注于风险评估而不生成操作。这种解耦让每个模型可以在自己擅长的方面做到最好，而不是让一个 LLM 扮演所有角色。

### Results

在 VisualWebArena 上达到 24.5% success rate（+1.8% over SOTA），在 Online-Mind2Web 上 16.0%（+1.3% over SOTA）。提升幅度不大但稳定。更重要的是定性分析：WAC 在高风险操作（如表单提交、账户设置更改）上的失败率显著低于 baseline，说明 consequence simulation + judge 机制确实在防止灾难性错误。

### Multi-Perspective Review

**🔬 Domain Expert**: 这篇论文把 world model 引入 web agent 是自然的延伸。之前的 web agent 工作（WebArena, SeeAct）主要关注 action generation，很少有人认真做 consequence prediction。但 world model 在这里是用 LLM 做的文本模拟，不是真正的环境模拟器——它的 prediction 质量完全依赖 LLM 对 web 交互的理解。

**🔧 Methodology Critic**: +1.8% 的提升在统计上可能不显著，尤其是 web agent 的 variance 很大。三模型协作增加了推理成本（每步要跑三个 LLM），但 paper 没有报告延迟和成本的 trade-off。另外 world model 的 simulation 质量如何评估？如果模拟是错的，judge 的审查也会出错。

**🔭 Visionary**: 这篇论文的核心 insight——"先在想象中试错，再在现实中执行"——是通用 agent 安全性的基础。如果 world model 的 simulation 质量能提升（比如用 browser rendering engine 做 ground truth），这个框架可以扩展到所有需要 safe exploration 的 agent 场景。

### Build-On Directions

1. **Learned web world model**: 用大量 web 交互数据训练专门的 web environment simulator，替代 LLM 做 consequence simulation
2. **Risk quantification**: 把 judge model 的二元判断（safe/risky）升级为连续风险评分，支持 risk-reward trade-off
3. **跨 CAW 的 connection**: WAC 用 LLM 做 web 环境的 world model，CAW 用 neural network 做 game 环境的 world model——两者都在解决同一个问题：让 agent 在行动前预测后果

### Trend Signal

Web agent 从"能做"走向"安全地做"。和 Curie 的 rigor 一样，agent 系统正在从 capability 阶段进入 reliability 阶段。

## 2. DeltaWorld: A Frame is Worth One Token — Efficient Generative World Modeling with Delta Tokens

- **Authors & Venue**: Tommie Kerssies, Gabriele Berton, Ju He, Qihang Yu, Wufei Ma, Daan de Geus, Gijs Dubbelman, Liang-Chieh Chen (Amazon, TU/e, JHU) — **CVPR 2026**
- **Links**: [arXiv](https://arxiv.org/abs/2604.04913) | [PDF](https://arxiv.org/pdf/2604.04913) | [Code](https://github.com/amazon-far)

### Core Problem

Video world model 面临一个根本矛盾：**discriminative model**（如 DINO-World）计算高效但只能输出确定性预测（隐式地 average over 所有可能的未来），**generative model**（如 Cosmos）能生成多样的未来但计算成本极高（几十亿参数）。这个矛盾的根源是 representation 的效率——现有方法用全帧 token（几百到几千个 token per frame）做生成，导致序列太长、计算太贵。

### Key Contribution

DeltaTok 提出了一个极致压缩的视频 tokenizer：不编码整帧，而是编码相邻帧的 VFM feature **差值**（delta），并且压缩到**一个 token per frame**。这是一个 ~1024x 的压缩。关键 insight 是：帧间变化比完整帧 compressible 得多——大部分像素不变，变化集中在运动区域。配合 Best-of-Many (BoM) 训练（采样 256 个候选，只监督最接近 ground truth 的那个），DeltaWorld 用 35x 更少的参数和 2000x 更少的 FLOPs 超越了 Cosmos-12B。

### Method

分两个模块。**DeltaTok**（tokenizer）：encoder 接收前一帧和当前帧的 VFM features（如 DINOv2），输出一个 delta token；decoder 用前一帧 features + delta token 重建当前帧。架构是 ViT blocks with self-attention。训练用简单的 L2 reconstruction loss。

**DeltaWorld**（predictor）：在 delta token 序列上做自回归预测。输入是历史 delta tokens + action（如果有），输出是下一个 delta token。用 Transformer with cross-attention，从 256 个 noise query embeddings 中采样多个候选未来。BoM 训练只 supervise 最优候选。

设计上最聪明的地方：delta formulation 提供了一个**自然的 prior**——"预测零 delta"等于"下一帧和上一帧一样"，这自动 recover 了 discriminative baseline 的 mean prediction。所以 generative 模型的 floor 就是 discriminative 的水平，上行空间全是 bonus。

### Results

在 VSPW、Cityscapes、KITTI 三个 dense forecasting benchmark 上，DeltaWorld 全面超越 Cosmos-12B（50.1 vs 47.7 mIoU on VSPW, 55.4 vs 53.3 on Cityscapes），同时参数少 35x、FLOPs 少 2000x。Ablation 清晰地展示了每个设计选择的贡献：delta 编码比全帧编码在 mean mIoU 上好 10 个点（45.5 vs 35.5），因为零 delta prior 提供了稳定的 baseline。

### Multi-Perspective Review

**🔬 Domain Expert**: 这是今年 world model 领域最 elegant 的论文之一。1 token per frame 的压缩率是一个 paradigm shift——从"如何生成高质量视频"转向"用最少的信息表示变化"。这和 information theory 的 rate-distortion 完美对应。BoM 训练虽然不是新方法（CVAE 时代就有），但在这个 context 下用得很自然。CVPR 2026 实至名归。

**🔧 Methodology Critic**: BoM 训练只优化最优候选，不能保证生成的 256 个候选覆盖了完整的未来分布——可能有些 mode 完全没被采样到。另外 delta token 对快速场景变化（如突然转弯、遮挡出现）可能不够用——变化大的帧用一个 token 很难编码。论文没有测试高速运动或突变场景。

**🔭 Visionary**: "用 delta 而不是 full state"这个思想可以扩展到很多领域：action delta（而不是 absolute action），graph delta（而不是 full graph）——等等，这不就是 CAW 的 affordance graph prediction 吗？CAW 预测的也是 graph 的变化（Δg），不是完整的 graph。DeltaTok 和 CAW 在抽象层面是同一个 idea：**压缩变化比压缩状态更高效**。

### Build-On Directions

1. **Hierarchical delta**: 对于突变帧，用 coarse delta（1 token）+ fine delta（多 token）的层级结构
2. **Action-conditioned DeltaWorld**: 加入 action conditioning，让 delta prediction 变成 action-conditioned，直接可以做 planning
3. **Delta + affordance**: 结合 DeltaTok 的 perceptual delta 和 CAW 的 affordance delta，一个管 pixel 变化，一个管 rule 变化

### Trend Signal

World model 从"大力出奇迹"（Cosmos 12B）走向"精巧设计"（DeltaWorld 35x smaller）。和 NLP 领域的 small model revival（Phi, Gemma）类似，说明好的 representation 比大参数更重要。

## 3. Action Images: End-to-End Policy Learning via Multiview Video Generation

- **Authors & Venue**: Haoyu Zhen, Zixian Gao, Qiao Sun, Yilin Zhao, Yuncong Yang, Yilun Du, Tsun-Hsuan Wang, Yi-Ling Qiao, Chuang Gan (UMass, Harvard, NVIDIA, Genesis AI) — arXiv 2604.06168
- **Links**: [arXiv](https://arxiv.org/abs/2604.06168) | [PDF](https://arxiv.org/pdf/2604.06168) | [Project](https://actionimages.github.io) | [Code](https://github.com/UMass-Embodied-AGI/ActionImages)

### Core Problem

Video generation model 做 world model 很强（Sora、Cosmos），但从 video 到 robot policy 之间有一个 gap：**video 的泛化不等于 policy 的泛化**。现有方法要么用额外的 action head 把 video features 映射到 action space（丢失了 video model 的泛化能力），要么用低维 action token（不 pixel-grounded，换视角就失效）。核心问题是：怎么让 video backbone 本身直接输出 policy，不需要额外的 action module？

### Key Contribution

Action Images 把 7-DoF 机器人 action 转换成**视觉化的 action image**——在每个相机视角下，把末端执行器的位置、朝向、夹爪状态渲染成 RGB 高斯热力图。这样 action 和 observation 共享同一个 pixel space，video model 可以同时生成未来画面和未来 action（作为"action video"的一部分）。不需要 policy head、不需要 action tokenizer——video backbone 本身就是 zero-shot policy。一个统一模型同时支持四种能力：video-action joint generation、action-conditioned video generation、video-to-action labeling、video-only generation。

### Method

Action 到 image 的转换分三步：(1) 把 7-DoF action（位置 + 旋转 + 夹爪）转换成三个 3D 语义点（position point, normal point, up point）；(2) 用已知的相机参数把 3D 点投影到每个视角的 2D 像素空间；(3) 把 2D 点渲染成 RGB 高斯热力图（R=position, G=normal, B=up+gripper）。反向解码时，从热力图找 2D 峰值，用多视角几何恢复 3D 坐标，再计算旋转。

模型基于 Wan 2.1-I2V-14B 视频生成 backbone，用 flow matching objective 训练。通过不同的 masking strategy 实现四种 task：85% 做 joint generation（mask video + action），5% 做 action-conditioned video generation，5% 做 video-to-action labeling，5% 做 video-only generation。训练数据混合 DROID（80K real trajectories）、RLBench（180K sim trajectories）、BridgeV2（30K real trajectories）。

关键设计选择：用 multi-view action image 而不是单视角，因为单视角有深度歧义——同一个 2D 点可能对应不同深度的 3D 位置。多视角交叉验证解决了这个问题。

### Results

Zero-shot success rate 碾压所有 baseline：在 RLBench 上 pick cup 30%（pi_0.5 只有 0%），reach target 60%（pi_0.5 5%），close drawer 50%（pi_0.5 35%）。在 real-world xArm 任务上也是最强。Video generation 质量同样 SOTA（PSNR 23.48 vs TesserAct 20.83）。Video-to-action labeling 超过 TAPIR 和 CoTracker3。

这些结果说明：**pixel-grounded action representation 是 video model 泛化到 policy 的关键**。不需要额外的 action module，video backbone 自己就能当 policy。

### Multi-Perspective Review

**🔬 Domain Expert**: 这是一个非常 elegant 的 idea——把 action 变成图片，让 video model 处理，省去了所有 cross-modal alignment 的麻烦。在 embodied AI 领域，之前有 RT-2 用 text token 表示 action，有 Octo 用 continuous token——Action Images 用 pixel 是最自然的选择，因为 video model 最擅长处理 pixel。Zero-shot 30-60% 的 success rate 在 manipulation 领域已经很强了。

**🔧 Methodology Critic**: 目前是 open-loop 的（一次性生成整个 action sequence），没有 closed-loop 反馈。实际部署需要高频 re-planning。推理延迟 2.3 秒对实时控制太慢。另外热力图的精度受限于图像分辨率——512x512 的图只有约 0.2% 的空间分辨率精度。对精细操作（如螺丝拧紧）可能不够。

**🔭 Visionary**: 如果这个 paradigm 成立，那 robot policy 的 scaling 就变成了 video model 的 scaling——用更大的 video model、更多的 video 数据就能得到更好的 policy。这比传统的 robot learning（需要专门收集 robot 数据）有更大的 scaling potential。

### Build-On Directions

1. **Closed-loop Action Images**: 每步生成短 horizon action image，执行后重新观测、重新生成，形成 MPC-style 的 closed-loop 控制
2. **Affordance Action Images**: 在 action image 上叠加 affordance map（哪些区域可以交互），让 model 同时理解"能做什么"和"该怎么做"
3. **Action Image 做 world model 的 planning**: 用 Action Images + video model 做 imagination-based planning——在想象中试不同的 action image，选最好的

### Trend Signal

Robotics 正在被 video generation 重新定义。从 RT-2（LLM as policy）到 Action Images（video model as policy），control 问题正在被转化为 generation 问题。这是"所有任务最终都变成 next-token prediction"的又一个实例。

## 4. D²-Align: Taming Preference Mode Collapse via Directional Decoupling Alignment

- **Authors & Venue**: Chubin Chen, Sujie Hu, Jiashu Zhu, Meiqi Wu, Jintao Chen, Yanxun Li, Nisha Huang, Chengyu Fang, Jiahong Wu, Xiangxiang Chu, Xiu Li — **CVPR 2026**
- **Links**: [arXiv](https://arxiv.org/abs/2512.24146) | [PDF](https://arxiv.org/pdf/2512.24146)

### Core Problem

用 RLHF 对齐 text-to-image diffusion model 时，会出现**偏好模式坍塌**（Preference Mode Collapse, PMC）：模型在 reward score 上越来越高，但生成的图像越来越单一——风格趋同、过度曝光、构图模板化。本质上是 reward hacking：模型找到了 reward model 的 bias 并无限放大，牺牲了多样性换取高分。这在文生图领域特别严重，因为"好看"的标准本身就有 bias（偏向高饱和度、高对比度、特定构图）。

### Key Contribution

D²-Align 提出了一个**方向解耦**框架来解决 PMC。核心思想：不是直接优化 reward model 的输出，而是先在 reward model 的 embedding space 里学一个**方向纠正**（directional correction），把 reward signal 中导致 mode collapse 的方向分离出来并抑制。这样模型可以朝"真正的质量提升"方向优化，而不是朝"reward model 的 bias"方向优化。Reward model 本身保持 frozen——纠正只发生在 embedding 层面。

另一个贡献是 **DivGenBench**——第一个专门量化 PMC 程度的 benchmark。之前的评估只看 reward score（越高越好），不看多样性（越高分图片越像就越有问题）。DivGenBench 同时衡量质量和多样性。

### Method

D²-Align 在 reward model 的 penultimate layer embedding 上做操作。训练一个轻量的方向纠正模块，学习把 embedding 分解为两个方向：(1) quality direction（和真正的图像质量相关）和 (2) bias direction（导致 mode collapse 的 reward model 偏好）。优化时只沿 quality direction 更新 diffusion model，抑制 bias direction 的梯度。

设计上的关键选择是**保持 reward model frozen**——不改 reward model 本身，只改用它的方式。这避免了 reward model retraining 的成本，也保证了纠正是 lightweight 和 plug-in 的。

### Results

D²-Align 在 DivGenBench 上同时提升了质量和多样性，而之前的 RLHF 方法（如 DDPO、DRaFT）只能提升质量但牺牲多样性。定性对比非常直观：baseline RLHF 生成的图片全是"高饱和度+中心构图+暖色调"，D²-Align 生成的图片保持了风格多样性的同时 reward score 也不低。

### Multi-Perspective Review

**🔬 Domain Expert**: PMC 是 RLHF for diffusion 领域公认的痛点，但之前大多数工作只是在 paper 里提一句"mode collapse is a concern"，没有系统性地定义和解决。D²-Align 第一次把 PMC 形式化，并且提出了一个优雅的解决方案。DivGenBench 作为 benchmark 的价值可能比方法本身更大——它让整个社区可以开始系统性地评估 diversity。

**🔧 Methodology Critic**: "Quality direction" vs "bias direction" 的分解依赖于训练数据和分解方法——如何保证学到的 quality direction 确实是 quality 而不是另一种 bias？另外评估 diversity 本身就很主观——DivGenBench 用什么 metric 量化多样性？如果用 FID diversity（生成图片之间的距离），这对 sample size 很敏感。

**🔭 Visionary**: 这个思路可以推广到所有 RLHF 场景——LLM 的 sycophancy（过度迎合用户）本质上也是 preference mode collapse。D²-Align 的 directional decoupling 原理可能对 LLM alignment 同样有用。

### Build-On Directions

1. **D²-Align for LLM**: 把方向解耦应用到 LLM RLHF，解决 sycophancy 和 reward hacking
2. **Adaptive direction learning**: 让方向纠正模块在线更新，随着 diffusion model 优化动态调整 bias direction
3. **Multi-reward composition**: 多个 reward model（美感、一致性、安全性）各自有不同的 bias direction，学习同时纠正多个方向

### Trend Signal

AI 生成模型的优化从"单维度最优"走向"多维度平衡"。之前是"reward 越高越好"，现在要同时考虑质量、多样性、安全性。这和 CAW 的 message 有共鸣——世界不是一维的，agent 的目标也不应该是一维的。
