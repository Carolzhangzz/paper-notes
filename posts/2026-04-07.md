# Paper Notes — 2026-04-07

## 1. Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models

- **Authors & Venue**: Kaijin Chen, Dingkang Liang, Xin Zhou, Yikang Ding, Xiaoqiang Liu, Pengfei Wan, Xiang Bai (HUST & Kling Team, Kuaishou Technology). arXiv 2026. [Paper](https://arxiv.org/abs/2603.25716) | [Code](https://github.com/H-EmbodVis/HyDRA) | [Project](https://kj-chen666.github.io/Hybrid-Memory-in-Video-World-Models/)

- **Core Problem**: 现有的 video world model 把环境当作静态画布来处理——它们能记住不动的背景，但当动态主体（行人、动物）离开摄像头视野后再回来时，就会出现主体冻结、扭曲甚至消失的问题。这背后的根本挑战是：世界模型需要在主体不可见的期间"脑补"它的运动轨迹，同时保持静态背景的一致性。这个问题对自动驾驶、embodied AI 等需要在动态环境中进行规划和决策的场景至关重要——一个看不见行人就忘记行人的世界模型是根本不可用的。

- **Key Contribution**: 论文提出了 **Hybrid Memory** 范式，要求模型同时扮演两个角色：静态背景的精确档案管理员 + 动态主体的警觉追踪器。核心创新包括三个层面：(1) 问题定义层面，首次形式化了 video world model 中的 hybrid memory 问题，明确区分了静态记忆和动态记忆的不同需求；(2) 数据层面，构建了 **HM-World** 数据集——59K 高保真视频片段（UE5 渲染，17 个场景，49 个主体），专门设计了 exit-entry 事件来训练和评估 hybrid memory 能力；(3) 架构层面，提出 **HyDRA**（Hybrid Dynamic Retrieval Attention），通过 memory tokenizer 压缩历史信息 + spatiotemporal affinity-driven 的 Top-K 检索机制，在保持 local denoising 稳定性的同时实现对隐藏主体的身份和运动连续性保持。

- **Method**: HyDRA 基于 Wan2.1-T2V-1.3B 构建，采用 Flow Matching 训练。架构有三个关键组件：首先是 **Camera Injection**，将相机位姿（旋转矩阵 R + 平移向量 t）展平为 12 维向量，通过 MLP 编码后与 latent feature 逐元素相加，让模型感知相机运动。其次是 **Memory Tokenizer**，用 3D 卷积（kernel 2x4x4）将历史帧的 latent 压缩为紧凑的 memory token，关键 insight 是对 latent 做时空交互后得到的表征远比原始 latent 更富含运动信息。最后是 **Dynamic Retrieval Attention**，替代标准 3D self-attention：对每个目标帧的 query，通过 spatial pooling + element-wise inner product 计算与所有 memory token 的时空亲和度，Top-K（K=10）检索最相关的 token，再与 local temporal window（窗口大小=5）的 key/value 拼接后做标准 attention。这种设计的好处是：local window 保证了去噪过程的稳定性，而 dynamic retrieval 让模型能跨越长时间跨度检索到离开视野的主体信息。

- **Results**: 在 HM-World 测试集（1000 样本，unseen 场景和主体）上，HyDRA 在所有指标上全面领先。PSNR 达到 20.357（vs Baseline 18.696、Context-as-Memory 18.921），新提出的 DSC（Dynamic Subject Consistency）指标 DSC_GT 达到 0.849。对比 DFoT（ICML 2025）会导致主体消失，Context-as-Memory（SIGGRAPH Asia 2025）会出现严重扭曲，商业模型 WorldPlay 虽能保持外观但运动会卡顿。消融实验表明 memory tokenizer 的时间维度至关重要（去掉后 PSNR 下降 1.281），以及 dynamic affinity retrieval 显著优于 FOV overlap retrieval（Subject Consistency 0.926 vs 0.908）。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 这篇工作精准地抓住了 video world model 的一个核心盲区——对 off-screen 动态实体的记忆和推理。Hybrid Memory 的问题定义很清晰，三个子挑战（spatiotemporal decoupling、out-of-view extrapolation、feature entanglement）的分析很到位。HM-World 数据集是重要贡献，填补了这个特定评估场景的空白。不过，数据集完全是合成的（UE5），泛化到真实世界的能力还有待验证。
  - 🔧 **Methodology Critic**: Memory Tokenizer 用 3D 卷积做压缩比较直接，可能丢失细粒度的外观信息（虽然消融实验证明了有效性）。Top-K retrieval 是 hard selection，可能会有 gradient 传播的问题。论文承认的 limitation——3 个以上主体时性能下降——暗示了 retrieval 机制在 multi-entity 场景下可能存在 attention 竞争问题。另外，训练只用了 10K iterations 在合成数据上，real-world 的 domain gap 可能很大。
  - 🔭 **Visionary**: 这篇工作的核心 insight——世界模型需要"imagination"能力来推断不可见实体的状态——指向了一个更深层的问题：真正的 world model 不应该只是 video prediction，而是需要具备 object-level 的状态追踪和物理推理能力。这和认知科学中的 object permanence 概念直接对应，可能启发更多 cognitive-inspired 的世界模型设计。

- **Build-On Directions**:
  1. **Real-world hybrid memory**: 在真实视频数据上训练和评估，可能需要结合 detection/tracking 模块来自动标注 exit-entry 事件，而不依赖合成数据。
  2. **Multi-entity scaling**: 解决 3+ 主体的性能瓶颈，可能需要 entity-level 的 memory bank 而不是全局的 token retrieval，类似 slot attention 的思路。
  3. **与 affordance/interaction 结合**: 当前方法只追踪主体的外观和运动，但不理解主体之间的交互。结合 affordance graph 或 interaction prediction 可以让 out-of-view extrapolation 更准确（比如预测两个人在视野外相遇时会发生什么）。

- **Trend Signal**: 这篇论文反映了 video world model 领域正在从"生成好看的视频"转向"真正理解世界动态"的趋势。Memory mechanism 成为核心瓶颈——不是怎么生成更多帧，而是怎么在长时间跨度上保持对世界状态的一致理解。这和 LLM 领域从"生成流畅文本"到"长上下文理解"的演进路径高度一致。

## 2. Mixture of Contexts for Long Video Generation (MoC)

- **Authors & Venue**: Shengqu Cai, Ceyuan Yang†, Lvmin Zhang, Yuwei Guo, Junfei Xiao, Ziyan Yang, Yinghao Xu, Zhenheng Yang, Alan Yuille, Leonidas Guibas, Maneesh Agrawala, Lu Jiang, Gordon Wetzstein (Stanford, ByteDance Seed, JHU, CUHK). **ICLR 2026**. [Paper](https://arxiv.org/abs/2508.21058) | [Project](https://primecai.github.io/moc/)

- **Core Problem**: 长视频生成的核心瓶颈是 **long context memory**——模型需要在长时间范围内保留和检索重要事件，同时不能出现 collapse 或 drift。然而 diffusion transformer 的 self-attention 是 O(n²) 的，当视频长度达到分钟级别（~180K tokens）时，显存和计算都变得不可承受。现有的 sparse attention 方案（sliding window、strided pattern）都是手工设计的，无法根据内容动态决定哪些历史帧是重要的。这个问题不仅限制了视频生成的长度，更根本地限制了模型学习长程依赖的能力。

- **Key Contribution**: 论文将长视频生成重新定义为 **internal information retrieval** 问题，提出 **Mixture of Contexts (MoC)**——一个 learnable sparse attention routing 模块，作为长期记忆检索引擎。MoC 的核心创新在于：每个 query token 动态选择少数几个 informative chunks + mandatory anchors（caption token、local window）来参与 attention 计算，而不是对所有历史 token 做 full attention。关键的是，这个 routing 是**非参数化但可学习的**——不添加额外的路由参数，而是通过端到端训练从大规模长视频数据中隐式学习路由策略。routing 使用 causal 约束防止时间环路。最终效果是：效率成为 retrieval 的**副产品**而非设计目标，模型自然地学会了只关注重要的历史信息，实现了近线性的计算扩展。

- **Method**: MoC 的核心思路是把长视频的 token 序列分成固定大小的 chunk，然后为每个 query token 学习一个 sparse routing decision——从所有历史 chunk 中选择最相关的几个来参与 attention。具体而言，routing 机制包含两个部分：(1) **mandatory anchors**，即 caption token 和 local temporal window，这些始终被选中，保证了文本条件的注入和局部时序连贯性；(2) **dynamic selection**，通过 query 和各个 chunk 的 key 之间的相似度来选择 top-k 个最相关的历史 chunk。这个路由器是非参数化的（不引入额外的 MLP 或 gating network），而是直接利用 attention 本身的 QK 相似度来做决策，因此可以通过标准的反向传播端到端优化。

训练策略上，作者采用了**渐进式稀疏化**（gradual sparsification）：先在 full attention 或较低稀疏度下让模型学会什么信息是重要的，然后逐步增加稀疏度，让模型在保持质量的同时学会更高效的检索。这种 curriculum 避免了一开始就施加强稀疏约束导致模型学不到长程依赖。causal routing 的约束确保了生成过程中的时序一致性——未来的 chunk 不能影响过去的生成。

- **Results**: 在 multi-shot 长视频生成（8-shot，约 1 分钟，~180K tokens）场景下，MoC 裁剪了约 85% 的 token pairs，将 FLOPs 降低了 7 倍，同时实现了分钟级别的上下文一致性（身份、场景、动作的跨镜头保持）。在 single-shot 短视频生成（8 秒，320x192，~6.5K tokens）场景下，MoC 裁剪了约 83% 的 token pairs，视频质量与 full attention 持平甚至略有提升——这表明 full attention 中大量的计算实际上是冗余的，甚至可能引入噪声。核心 takeaway 是：效率和质量不是 trade-off 关系，好的 retrieval 本身就能带来更好的生成。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 把长视频生成重新框架化为 information retrieval 问题是非常 elegant 的 insight。与现有的 sparse attention 方法（Longformer 的 sliding window、BigBird 的 random + global tokens）相比，MoC 的 learned routing 能够根据内容自适应地分配注意力，这在理论上更合理。被 ICLR 2026 接收说明了社区对这个方向的认可。作者团队（Stanford + ByteDance Seed）的资源优势使得大规模长视频训练成为可能，这是很多学术组难以复现的。
  - 🔧 **Methodology Critic**: 非参数化路由虽然 elegant，但也意味着路由能力完全依赖于 attention head 本身的表征质量——如果某些 head 的 QK space 没有很好地编码时序相关性，routing 就会失效。渐进式稀疏化的 curriculum 设计可能需要精心调参（何时开始稀疏、稀疏化速度等），论文中这部分的 ablation 细节值得关注。另外，~180K tokens 虽然达到了分钟级，但距离真正的长视频（10 分钟+）还有差距，near-linear scaling 的实际表现需要在更长序列上验证。
  - 🔭 **Visionary**: MoC 的 insight 可能远超视频生成本身。在任何需要处理超长序列的 transformer 模型中（长文档、蛋白质序列、代码库），learned sparse routing 都可能替代手工设计的 sparse pattern。更深层地看，这种"让模型自己学会什么值得记住"的范式，本质上是在让模型学习一种 **attention-based working memory**——这和认知科学中 working memory 的 capacity-limited、relevance-driven 特性高度一致。对于 video world model 而言，MoC 直接解决了长 rollout 的计算瓶颈，是实现分钟级甚至小时级世界模拟的关键 enabling technology。

- **Build-On Directions**:
  1. **MoC + World Model action conditioning**: 将 MoC 的 learned routing 应用到 action-conditioned video world model 中，让模型在长 rollout 时动态检索与当前 action 最相关的历史状态，可能显著提升 long-horizon planning 的质量。
  2. **Hierarchical MoC**: 当前的 chunk-level routing 是单层的，可以引入多层级的检索——先在 scene 级别粗筛，再在 frame 级别精选，类似 retrieval 中的 coarse-to-fine 策略，进一步提升超长视频的效率。
  3. **Cross-modal routing**: 将 MoC 的 routing 机制扩展到 multi-modal 场景——比如在 text-video-audio 联合生成中，让模型学习跨模态的 sparse attention pattern，决定什么时候需要从文本描述中检索信息、什么时候需要从音频中检索。

- **Trend Signal**: MoC 反映了一个重要的范式转变：从"设计更好的 attention pattern"到"让模型学习自己的 attention pattern"。这和 LLM 领域 Mixture of Experts 的思路异曲同工——不是给所有 token 分配相同的计算，而是根据内容动态分配。更宏观地看，这反映了 AI 领域从 hand-crafted inductive bias 向 learned inductive bias 的持续演进。在视频生成领域，这预示着未来的模型不会靠堆更多的 GPU 来暴力 scale attention，而是通过更智能的 information routing 来实现更长、更一致的生成。
