# Paper Notes — 2026-04-07

## 1. Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models

- **Authors & Venue**: Kaijin Chen, Dingkang Liang, Xin Zhou, Yikang Ding, Xiaoqiang Liu, Pengfei Wan, Xiang Bai (HUST & Kling Team, Kuaishou Technology). arXiv 2026. [Paper](https://arxiv.org/abs/2603.25716) | [Code](https://github.com/H-EmbodVis/HyDRA) | [Project](https://kj-chen666.github.io/Hybrid-Memory-in-Video-World-Models/)

- **Core Problem**: 现有的 video world model 把环境当作静态画布来处理——它们能记住不动的背景，但当动态主体（行人、动物）离开摄像头视野后再回来时，就会出现主体冻结、扭曲甚至消失的问题。这背后的根本挑战是：世界模型需要在主体不可见的期间"脑补"它的运动轨迹，同时保持静态背景的一致性。这个问题对自动驾驶、embodied AI 等需要在动态环境中进行规划和决策的场景至关重要——一个看不见行人就忘记行人的世界模型是根本不可用的。

- **Key Contribution**: 论文提出了 **Hybrid Memory** 范式，要求模型同时扮演两个角色：静态背景的精确档案管理员 + 动态主体的警觉追踪器。核心创新包括三个层面：(1) 问题定义层面，首次形式化了 video world model 中的 hybrid memory 问题，明确区分了静态记忆和动态记忆的不同需求；(2) 数据层面，构建了 **HM-World** 数据集——59K 高保真视频片段（UE5 渲染，17 个场景，49 个主体），专门设计了 exit-entry 事件来训练和评估 hybrid memory 能力；(3) 架构层面，提出 **HyDRA**（Hybrid Dynamic Retrieval Attention），通过 memory tokenizer 压缩历史信息 + spatiotemporal affinity-driven 的 Top-K 检索机制，在保持 local denoising 稳定性的同时实现对隐藏主体的身份和运动连续性保持。

- **Method**: HyDRA 基于 Wan2.1-T2V-1.3B 构建，采用 Flow Matching 训练。架构有三个关键组件：首先是 **Camera Injection**，将相机位姿（旋转矩阵 R + 平移向量 t）展平为 12 维向量，通过 MLP 编码后与 latent feature 逐元素相加，让模型感知相机运动。其次是 **Memory Tokenizer**，用 3D 卷积（kernel 2x4x4）将历史帧的 latent 压缩为紧凑的 memory token，关键 insight 是对 latent 做时空交互后得到的表征远比原始 latent 更富含运动信息。最后是 **Dynamic Retrieval Attention**，替代标准 3D self-attention：对每个目标帧的 query，通过 spatial pooling + element-wise inner product 计算与所有 memory token 的时空亲和度，Top-K（K=10）检索最相关的 token，再与 local temporal window（窗口大小=5）的 key/value 拼接后做标准 attention。这种设计的好处是：local window 保证了去噪过程的稳定性，而 dynamic retrieval 让模型能跨越长时间跨度检索到离开视野的主体信息。

- **Results**: 在 HM-World 测试集（1000 样本，unseen 场景和主体）上，HyDRA 在所有指标上全面领先。PSNR 达到 20.357（vs Baseline 18.696、Context-as-Memory 18.921），新提出的 DSC（Dynamic Subject Consistency）指标 DSC_GT 达到 0.849。对比 DFoT（ICML 2025）会导致主体消失，Context-as-Memory（SIGGRAPH Asia 2025）会出现严重扭曲，商业模型 WorldPlay 虽能保持外观但运动会卡顿。消融实验表明 memory tokenizer 的时间维度至关重要（去掉后 PSNR 下降 1.281），以及 dynamic affinity retrieval 显著优于 FOV overlap retrieval（Subject Consistency 0.926 vs 0.908）。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 这篇工作精准地抓住了 video world model 的一个核心盲区——对 off-screen 动态实体的记忆和推理。Hybrid Memory 的问题定义很清晰，三个子挑战（spatiotemporal decoupling、out-of-view extrapolation、feature entanglement）的分析很到位。HM-World 数据集是重要贡献，填补了这个特定评估场景的空白。不过，数据集完全是合成的（UE5），泛化到真实世界的能力还有待验证。
  - 🔧 **Methodology Critic**: Memory Tokenizer 用 3D 卷积做压缩比较直接，可能丢失细粒度的外观信息（虽然消融实验证明了有效性）。Top-K retrieval 是 hard selection，可能会有 gradient 传播的问题。论文承认的 limitation——3 个以上主体时性能下降——暗示了 retrieval 机制在 multi-entity 场景下可能存在 attention 竞争问题。另外，训练只用了 10K iterations 在合成数据上，real-world 的 domain gap 可能很大。
  - 🔭 **Visionary**: 这篇工作的核心 insight——世界模型需要"imagination"能力来推断不可见实体的状态——指向了一个更深层的问题：真正的 world model 不应该只是 video prediction，而是需要具备 object-level 的状态追踪和物理推理能力。这和认知科学中的 object permanence 概念直接对应，可能启发更多 cognitive-inspired 的世界模型设计。

- **Build-On Directions**:
  1. **Real-world hybrid memory**: 在真实视频数据上训练和评估，可能需要结合 detection/tracking 模块来自动标注 exit-entry 事件，而不依赖合成数据。
  2. **Multi-entity scaling**: 解决 3+ 主体的性能瓶颈，可能需要 entity-level 的 memory bank 而不是全局的 token retrieval，类似 slot attention 的思路。
  3. **与 affordance/interaction 结合**: 当前方法只追踪主体的外观和运动，但不理解主体之间的交互。结合 affordance graph 或 interaction prediction 可以让 out-of-view extrapolation 更准确（比如预测两个人在视野外相遇时会发生什么）。

- **Trend Signal**: 这篇论文反映了 video world model 领域正在从"生成好看的视频"转向"真正理解世界动态"的趋势。Memory mechanism 成为核心瓶颈——不是怎么生成更多帧，而是怎么在长时间跨度上保持对世界状态的一致理解。这和 LLM 领域从"生成流畅文本"到"长上下文理解"的演进路径高度一致。

## 2. Mixture of Contexts for Long Video Generation (MoC)

- **Authors & Venue**: Shengqu Cai, Ceyuan Yang†, Lvmin Zhang, Yuwei Guo, Junfei Xiao, Ziyan Yang, Yinghao Xu, Zhenheng Yang, Alan Yuille, Leonidas Guibas, Maneesh Agrawala, Lu Jiang, Gordon Wetzstein (Stanford, ByteDance Seed, JHU, CUHK). **ICLR 2026**. [Paper](https://arxiv.org/abs/2508.21058) | [Project](https://primecai.github.io/moc/)

- **Core Problem**: 长视频生成的核心瓶颈是 **long context memory**——模型需要在长时间范围内保留和检索重要事件，同时不能出现 collapse 或 drift。然而 diffusion transformer 的 self-attention 是 O(n²) 的，当视频长度达到分钟级别（~180K tokens）时，显存和计算都变得不可承受。现有的 sparse attention 方案（sliding window、strided pattern）都是手工设计的，无法根据内容动态决定哪些历史帧是重要的。这个问题不仅限制了视频生成的长度，更根本地限制了模型学习长程依赖的能力。

- **Key Contribution**: 论文将长视频生成重新定义为 **internal information retrieval** 问题，提出 **Mixture of Contexts (MoC)**——一个 learnable sparse attention routing 模块，作为长期记忆检索引擎。MoC 的核心创新在于：每个 query token 动态选择少数几个 informative chunks + mandatory anchors（caption token、local window）来参与 attention 计算，而不是对所有历史 token 做 full attention。关键的是，这个 routing 是**非参数化但可学习的**——不添加额外的路由参数，而是通过端到端训练从大规模长视频数据中隐式学习路由策略。routing 使用 causal 约束防止时间环路。最终效果是：效率成为 retrieval 的**副产品**而非设计目标，模型自然地学会了只关注重要的历史信息，实现了近线性的计算扩展。

- **Method**: MoC 的核心思路是把长视频的 token 序列分成固定大小的 chunk，然后为每个 query token 学习一个 sparse routing decision——从所有历史 chunk 中选择最相关的几个来参与 attention。具体而言，routing 机制包含两个部分：(1) **mandatory anchors**，即 caption token 和 local temporal window，这些始终被选中，保证了文本条件的注入和局部时序连贯性；(2) **dynamic selection**，通过 query 和各个 chunk 的 key 之间的相似度来选择 top-k 个最相关的历史 chunk。这个路由器是非参数化的（不引入额外的 MLP 或 gating network），而是直接利用 attention 本身的 QK 相似度来做决策，因此可以通过标准的反向传播端到端优化。

训练策略上，作者采用了**渐进式稀疏化**（gradual sparsification）：先在 full attention 或较低稀疏度下让模型学会什么信息是重要的，然后逐步增加稀疏度，让模型在保持质量的同时学会更高效的检索。这种 curriculum 避免了一开始就施加强稀疏约束导致模型学不到长程依赖。causal routing 的约束确保了生成过程中的时序一致性——未来的 chunk 不能影响过去的生成。

- **Results**: 在 multi-shot 长视频生成（8-shot，约 1 分钟，~180K tokens）场景下，MoC 裁剪了约 85% 的 token pairs，将 FLOPs 降低了 7 倍，同时实现了分钟级别的上下文一致性（身份、场景、动作的跨镜头保持）。在 single-shot 短视频生成（8 秒，320x192，~6.5K tokens）场景下，MoC 裁剪了约 83% 的 token pairs，视频质量与 full attention 持平甚至略有提升——这表明 full attention 中大量的计算实际上是冗余的，甚至可能引入噪声。核心 takeaway 是：效率和质量不是 trade-off 关系，好的 retrieval 本身就能带来更好的生成。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 把长视频生成重新框架化为 information retrieval 问题是非常 elegant 的 insight。与现有的 sparse attention 方法（Longformer 的 sliding window、BigBird 的 random + global tokens）相比，MoC 的 learned routing 能够根据内容自适应地分配注意力，这在理论上更合理。被 ICLR 2026 接收说明了社区对这个方向的认可。作者团队（Stanford + ByteDance Seed）的资源优势使得大规模长视频训练成为可能，这是很多学术组难以复现的。
  - 🔧 **Methodology Critic**: 非参数化路由虽然 elegant，但也意味着路由能力完全依赖于 attention head 本身的表征质量——如果某些 head 的 QK space 没有很好地编码时序相关性，routing 就会失效。渐进式稀疏化的 curriculum 设计可能需要精心调参（何时开始稀疏、稀疏化速度等），论文中这部分的 ablation 细节值得关注。另外，~180K tokens 虽然达到了分钟级，但距离真正的长视频（10 分钟+）还有差距，near-linear scaling 的实际表现需要在更长序列上验证。
  - 🔭 **Visionary**: MoC 的 insight 可能远超视频生成本身。在任何需要处理超长序列的 transformer 模型中（长文档、蛋白质序列、代码库），learned sparse routing 都可能替代手工设计的 sparse pattern。更深层地看，这种"让模型自己学会什么值得记住"的范式，本质上是在让模型学习一种 **attention-based working memory**——这和认知科学中 working memory 的 capacity-limited、relevance-driven 特性高度一致。对于 video world model 而言，MoC 直接解决了长 rollout 的计算瓶颈，是实现分钟级甚至小时级世界模拟的关键 enabling technology。

- **Build-On Directions**:
  1. **MoC + World Model action conditioning**: 将 MoC 的 learned routing 应用到 action-conditioned video world model 中，让模型在长 rollout 时动态检索与当前 action 最相关的历史状态，可能显著提升 long-horizon planning 的质量。
  2. **Hierarchical MoC**: 当前的 chunk-level routing 是单层的，可以引入多层级的检索——先在 scene 级别粗筛，再在 frame 级别精选，类似 retrieval 中的 coarse-to-fine 策略，进一步提升超长视频的效率。
  3. **Cross-modal routing**: 将 MoC 的 routing 机制扩展到 multi-modal 场景——比如在 text-video-audio 联合生成中，让模型学习跨模态的 sparse attention pattern，决定什么时候需要从文本描述中检索信息、什么时候需要从音频中检索。

- **Trend Signal**: MoC 反映了一个重要的范式转变：从"设计更好的 attention pattern"到"让模型学习自己的 attention pattern"。这和 LLM 领域 Mixture of Experts 的思路异曲同工——不是给所有 token 分配相同的计算，而是根据内容动态分配。更宏观地看，这反映了 AI 领域从 hand-crafted inductive bias 向 learned inductive bias 的持续演进。在视频生成领域，这预示着未来的模型不会靠堆更多的 GPU 来暴力 scale attention，而是通过更智能的 information routing 来实现更长、更一致的生成。

## 3. When Narrower is Better: The Narrow Width Limit of Bayesian Parallel Branching Neural Networks

- **Authors & Venue**: Zechen Zhang, Haim Sompolinsky (Harvard University). **ICLR 2025**. [Paper](https://arxiv.org/abs/2407.18807)

- **Core Problem**: 神经网络领域有一个被广泛接受的常识："越宽越好"——更宽的网络泛化性能更强。在无穷宽极限下，神经网络收敛为 Gaussian Process（NNGP），此时 kernel 是 task-independent 的，无法根据数据结构进行自适应。这篇论文挑战了这一认知，提出一个根本性的问题：如果我们往**相反方向**走——让 parallel branching network 变**窄**而不是变宽——会发生什么？这个问题重要是因为它揭示了传统 "wider is better" 直觉并不普适，并为网络架构设计提供了新的理论指导。

- **Key Contribution**: 论文在 Bayesian Parallel Branching Neural Network (BPB-NN，类似带 residual block 的网络) 上定义并分析了一个全新的 **narrow-width regime**。核心发现是：当 BPB-NN 的宽度远小于训练样本数时，各个 branch 会发生 **symmetry breaking**——不同 branch 的 readout norm 会根据训练数据的结构自适应调整，从而实现 task-dependent 的 kernel renormalization。在 bias-limited 场景下，narrow width 的性能普遍**优于或等于** wide width。论文还证明了一个 Equipartition Conjecture：在 student-teacher setup 的极端 narrow limit 下，student 的 branch readout norm 会自动收敛到 teacher 的 branch norm，意味着网络自动发现了正确的分支权重分配。

- **Method**: 作者构建了一个 L-branch 的 BPB-NN，每个 branch 独立操作（无权重共享），对图节点特征施加不同阶的邻接矩阵幂 A^l。在 Bayesian 框架下，积分掉 readout weights 后得到一个有效 Hamiltonian，其中关键 order parameter u_l 表示每个 branch 的 readout norm 的统计平均。整体 kernel 变为 K = Σ(1/L) u_l K_l。核心理论分析表明：在无穷宽 GP 极限（α = P/N → 0），所有 u_l 退化为相同的常数 σ_w²——完全失去了 task adaptation 能力；而在 narrow width（α → ∞），entropic 和 energetic terms 的平衡发生变化，导致 u_l 根据训练数据发生分化，实现了 symmetry breaking。作者通过 saddle point equations 严格推导了这一转变，并用 Hamiltonian Monte Carlo (HMC) 采样验证理论预测。

- **Results**: 在 Contextual Stochastic Block Model (CSBM) 合成图数据上，N=4（narrow）时 student branch norm 精确匹配 teacher norm（理论预测完美吻合），而 N=1024（wide）时 student norm 退化为 GP 极限。在 Cora 引用网络上，bias 随宽度增加单调增长趋向 GP regime；narrow width 时 branch readout norm 独立于 σ_w 收敛，高阶卷积 branch（更大的 l）贡献更多，揭示了数据集需要高阶特征交互。存在 width-dependent 的泛化误差最小值，在某些 σ_w 下中等宽度最优。结果扩展到 residual-MLP 和 CIFAR-10，证明 narrow-width 效应是 branching network 的普遍特性。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 这是一篇扎实的理论工作，发表在 ICLR 2025 是合理的。挑战 "wider is better" 的 conventional wisdom 需要勇气和严谨的数学证明。与 NTK/NNGP 理论（Jacot et al., Lee et al.）的联系清晰，symmetry breaking 的分析优雅。Sompolinsky 是统计物理和神经科学的大佬，这种 physics-inspired 的分析方法在 ML 理论中独树一帜。
  - 🔧 **Methodology Critic**: 整个分析在 Bayesian posterior（lazy regime）下进行，对应 fully trained over-parameterized networks 的集成，这限制了向 SGD 训练的实际网络的迁移性。论文自己承认在 variance-limited regime（大多数真实数据集）中 narrow-width 优势减弱。Feature learning 的 rich regime 下结果是否成立是一个开放问题。没有与标准 GCN 训练方法做外部对比，所有比较都是内部的。
  - 🔭 **Visionary**: 这篇工作的深层意义在于：网络架构中的 "size" 不是越大越好——关键是让网络的容量与数据结构匹配。Narrow width 作为一种隐式正则化器，迫使网络学会更高效地分配 branch 权重。这可能启发一种新的架构搜索范式：不是搜最大的，而是搜与数据结构最匹配的宽度。对 GNN 领域尤其有启发——不同阶的 message passing 应该有不同的权重，而 narrow width 自动学会了这个分配。

- **Build-On Directions**:
  1. **Rich regime 扩展**: 将 narrow-width 分析从 Bayesian/lazy regime 扩展到 feature learning 的 rich regime，可能需要 mean-field theory 或 tensor program 框架的工具。
  2. **Adaptive width architectures**: 基于 narrow-width 理论，设计能自动调整各 branch 宽度的架构——让数据决定哪些 branch 需要更多或更少的参数。
  3. **Graph-specific applications**: 在 heterogeneous graph 或 knowledge graph 上验证 narrow-width 效应，不同阶的邻居信息可能有天然不同的重要性。

- **Trend Signal**: 反映了 ML 理论从"追求极限"（无穷宽、无穷深）转向"理解有限资源约束"的趋势。在大模型时代，理解什么时候小/窄模型更好具有直接的实际价值——不是所有问题都需要 billion-parameter 模型。

## 4. *New News*: System-2 Fine-tuning for Robust Integration of New Knowledge

- **Authors & Venue**: Core Francisco Park, Zechen Zhang, Hidenori Tanaka (Harvard University). arXiv 2025. [Paper](https://arxiv.org/abs/2505.01812)

- **Core Problem**: LLM 能够通过 in-context learning (ICL) 有效使用新信息，但通过 fine-tuning 将同样的信息真正"内化"到权重中却非常困难——存在巨大的 **FT-ICL gap**。这个问题很重要，因为 ICL 受限于上下文窗口大小和推理计算成本，而对于需要永久集成的知识（更新的 API、新的科学发现、变化的世界事件），weight-based learning 是必需的。现有的 naive fine-tuning 在知识内化上远不如简单地把信息放进 context window。

- **Key Contribution**: 论文引入了 **New News** 数据集——75 条横跨数学、编程、发现、排行榜、事件五个领域的假设性但合理的新闻，配有 375 个下游评估问题。核心方法创新是 **System-2 Fine-tuning (Sys2-FT)**：通过 self-play data generation 将模型在有 context 时处理的知识蒸馏到模型权重中。三种协议中 **Self-QA** 最有效——先让模型在有 context 的情况下生成问题和答案，然后用这些 QA pairs（去掉原始 context）做 fine-tuning。论文还发现了 **contextual shadowing effect**：如果训练数据中包含原始新闻作为 context 前缀，学习效果会灾难性下降，因为模型可以直接通过 ICL 回答正确，答案 token 的 loss gradient 变得微不足道。

- **Method**: Sys2-FT 的核心思路借鉴了认知科学中的 memory consolidation：给模型新闻作为 context，让它生成衍生训练数据，然后在没有原始新闻的情况下用这些合成数据做 fine-tuning。Self-QA 协议分两阶段：(a) 模型在有 context 时生成 10 个关于新闻的问题；(b) 模型在有 context 时生成这些问题的答案。产生的 QA pairs 成为训练数据（关键：不包含原始新闻作为 context 前缀）。每条新闻生成 1024 个对话，使用 LoRA（r=16, alpha=32）在 Qwen 2.5 系列模型（0.5B 到 32B）上训练。训练策略采用 constant LR=1e-4，4 epochs，保存 80 个 checkpoint 用于分析学习动态。

- **Results**: Self-QA 在所有领域和模型规模上显著优于 Paraphrase 和 Implication 协议。在 math 和 coding 领域，Self-QA 在 14B+ 模型上几乎匹配 ICL 性能。Quantitative 领域（math, coding）受益最大，discoveries/leaderboards/events 改进不太稳定。通用能力保持良好：MMLU-Pro、GSM8K、IFEval 分数基本不变。更大的模型从 Sys2-FT 受益更多（emerging scaling law）。Continual learning 实验显示跨领域遗忘较小。还发现了 "curse of overexposure"——weight-based 学习提升时，同一新闻的 ICL 能力可能暂时下降。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: FT-ICL gap 是 LLM 社区关注的重要问题，这篇工作提供了一个干净的实验框架来系统研究它。Contextual shadowing effect 是一个有价值的 negative finding——它解释了为什么很多 naive knowledge injection 方法效果不好，对实践有直接指导意义。与 knowledge editing（ROME、MEMIT）和 continual learning 文献有清晰的关系但取了不同的 angle。
  - 🔧 **Methodology Critic**: 数据集规模较小（75 条新闻），是否能反映大规模知识更新的挑战值得质疑。"假设性但合理的新闻" 回避了与模型预训练知识冲突的问题——真实场景中的知识更新往往需要覆盖旧知识，这是更难的。Scaling law 的证据还比较初步（只在 Qwen 2.5 一个系列上验证）。Self-QA 生成的训练数据质量完全依赖于模型本身的推理能力，对弱模型可能不适用。
  - 🔭 **Visionary**: Sys2-FT 的 "System-2" 命名暗示了一个更深的 vision：让模型不仅记住事实，而是通过"推理"来内化知识——类似人类的深度学习过程。如果这个方向成功 scale，可能改变 LLM 更新知识的方式——从简单的 continued pretraining 转向更结构化的知识蒸馏。Contextual shadowing 也暗示了一个更一般的原则：训练数据中不应该给模型"捷径"，否则模型会走捷径而不是真正学习。

- **Build-On Directions**:
  1. **Conflicting knowledge update**: 将 Sys2-FT 扩展到需要覆盖旧知识的场景——比如"Python 3.15 改变了 dict 的默认行为"，模型需要 unlearn 旧行为并学习新行为。
  2. **Multi-hop reasoning over injected knowledge**: 测试 Sys2-FT 注入的知识能否支持多步推理——比如注入两条新闻后，模型能否回答需要组合两条新闻信息的问题。
  3. **Sys2-FT for domain adaptation**: 将 self-play 数据生成的思路应用到专业领域适配——比如让医学 LLM 通过 Self-QA 内化新的临床指南。

- **Trend Signal**: 反映了 LLM 领域从"如何训练更大的模型"到"如何高效更新现有模型知识"的转变。随着 LLM 部署越来越广泛，knowledge update 成为核心基础设施问题。这篇工作提供了一个介于 RAG（外部记忆）和 full retraining（成本过高）之间的中间方案。

## 5. Linear Latent World Models in Simple Transformers: A Case Study on Othello-GPT

- **Authors & Venue**: Dean S. Hazineh, Zechen Zhang, Jeffery Chiu (Harvard University). **NeurIPS 2023 SoLaR Workshop**. [Paper](https://arxiv.org/abs/2310.07582) | [Code](https://github.com/DeanHazineh/Emergent-World-Representations-Othello)

- **Core Problem**: Foundation models 到底是真正"理解"了它们训练的领域，还是只是在做高级的"随机模仿"（stochastic mimicry）？这是 AI 和 mechanistic interpretability 领域最核心的辩论之一。通过使用 Othello（黑白棋）这个完全可观测、有 ground truth 的受控环境，论文提供了严格的证据来回答这个问题。

- **Key Contribution**: 论文在 Othello-GPT 上做出了四个关键发现：(1) 提出 **Mine/Yours/Empty** 编码方案（相对于当前玩家而非绝对 Black/White），发现模型内部正是用这种博弈论上更自然的表征来编码棋盘状态，线性探针准确率从 75% 飙升到 99.5%；(2) 设计了 **因果干预框架**——通过逆线性探针直接操纵 activation，强制模型基于虚假棋盘状态做预测，证明线性世界表征是被因果地使用的（不只是附带产物）；(3) 发现因果利用集中在中间层（4-5 层），最后两层虽然探针准确率最高但没有因果效应——representation quality 和 causal utility 的解离令人惊讶；(4) 识别了四类 attention head（Last Token / First Token / Mine Token / Yours Token），揭示了世界模型的具体电路结构。

- **Method**: 实验在三种规模的 Othello-GPT（1L1H、4L8H、8L8H）上进行。线性探针训练一个 R^512 → R^(64×3) 的线性映射，对每个棋盘位置做 Mine/Yours/Empty 三分类。因果干预通过逆线性探针将目标（虚假）棋盘状态编码回 activation space，替换真实的 activation，然后观察模型的下一步预测是否改变。如果模型的预测跟随了虚假棋盘状态，就说明该层的线性表征被因果地使用了。Attention head 分类通过分析每个 head 的注意力权重分布来实现。

- **Results**: Mine/Yours 编码在第 5 层达到 ~99.5% 准确率（vs Black/White 的 ~75%），证明模型发现了博弈论上正确的抽象。因果干预在中间层（4-5 层）最有效，最后两层无因果效应。1 层模型编码了线性表征但不因果使用它——暗示存在替代决策机制。4 层和 8 层模型展现相似的因果模式。早期游戏阶段的干预效果强于后期。即使最小的 1L1H 模型也达到了 94.9% 的合法落子准确率。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 这篇工作是 Li et al. (2022) "Emergent World Representations" 和 Neel Nanda 分析的重要扩展。Mine/Yours 表征的发现非常优雅——模型自主发现了博弈论上有意义的抽象，而不是人类直觉上的 Black/White 编码。因果干预（不只是相关性分析）的方法论提升了证据强度。发表在 NeurIPS workshop 级别适当。
  - 🔧 **Methodology Critic**: Othello 是一个完全可观测的确定性两人博弈——泛化到自然语言或更复杂领域的证据不足。因果干预依赖线性假设——如果模型使用了非线性表征，干预会低估因果性。1 层模型的悖论（有表征但不因果使用）没有完全解决。
  - 🔭 **Visionary**: 这篇工作的深层意义在于：transformer 确实能在训练过程中自动发现有意义的世界模型，而且是以可解释的线性形式。这为 mechanistic interpretability 提供了希望——如果我们能在 LLM 中找到类似的线性世界表征，就可能理解模型到底"知道"什么。Mine/Yours 的发现暗示：模型发现的抽象可能比人类预期的更"正确"——我们不应该用人类直觉来限定 interpretability 的搜索空间。

- **Build-On Directions**:
  1. **扩展到更复杂的游戏**: 在 Chess、Go 或 StarCraft 上寻找类似的线性世界表征，测试 complexity 增加后线性表征是否仍然存在。
  2. **LLM 中的线性世界模型**: 在语言模型中寻找类似的 Mine/Yours 式的 relative representation——比如在对话中，模型是否用 speaker-relative 而非 absolute 的方式编码信息。
  3. **因果干预用于 model editing**: 将逆线性探针技术发展为一种精准的 model editing 工具——不改权重，只改 activation，实现对模型行为的精细控制。

- **Trend Signal**: 反映了 mechanistic interpretability 领域从"模型内部有 feature"到"这些 feature 被因果地使用"的方法论进步。也反映了用 controlled environment（游戏、合成数据）来严格研究 LLM 内部机制的趋势——比在自然语言上做 interpretability 更干净，虽然泛化性存疑。

## 6. Memorization and Knowledge Injection in Gated LLMs (MEGa)

- **Authors & Venue**: Xu Pan, Ely Hahami, Zechen Zhang, Haim Sompolinsky (Harvard University). arXiv 2025. [Paper](https://arxiv.org/abs/2504.21239)

- **Core Problem**: LLM 无法在不发生 catastrophic forgetting 的情况下持续添加新记忆和整合新知识——这与人类终身学习的能力形成鲜明对比。现有方案要么依赖大 context window（相当于 working memory 而非 long-term storage），要么用 RAG（外部记忆缓冲，不修改模型内部知识），要么 fine-tuning（严重遗忘）。这个问题对需要持续更新知识的 AI agent 至关重要——一个忘记之前交互的 agent 无法建立长期关系或积累经验。

- **Key Contribution**: 论文提出 **MEGa (Memory Embedded in Gated LLMs)**——一个受人脑互补记忆系统启发的 continual learning 框架。核心创新：每条记忆存储在一组独立的 gated low-rank weights（LoRA adapter）中，推理时通过 gating 机制根据 query embedding 与存储的 memory embedding 的相似度来激活相关的记忆权重。这样每条记忆完全独立存储，彻底避免了记忆间的干扰。论文还提出了 **iRAG (internal RAG)**——两阶段推理，先用激活的 LoRA 重建完整记忆，再基于重建的记忆回答问题，性能接近外部 RAG。生物学灵感来自海马体（快速学习器 = gated LoRA）和皮层（慢速学习器 = base weights）的互补系统。

- **Method**: 基于 Llama-3.1-8B-Instruct 构建。Fine-tuning 阶段：对每条新记忆，计算并存储其 context key（最后一个 MLP 层输入处的 activation 平均值），然后训练一个独立的 LoRA adapter（r=128，target 所有 MLP 层）。推理阶段：所有 LoRA adapter 通过加权求和组合，gating weights 由 query embedding 和所有 context keys 的 softmax 相似度决定（β=1 常规查询，β=0.1 组合性问题）。三种推理模式：Memory Recall（重建完整记忆）、QA（直接回答）、iRAG（先 recall 再回答）。每条记忆用 9 个 paraphrase 做数据增强。

- **Results**: 在 Fictional Characters（50 条记忆）和 Wikipedia Events（1000 条采样）数据集上，MEGa 在 50 条连续记忆后 QA 准确率分别达到 72.53% 和 78.03%，而所有 continual fine-tuning baseline（Full、LoRA、EWC、L2）都严重遗忘（最高 17.9%）。MEGa 的 gate 选择准确率达 85-87.8%。iRAG 进一步提升到 80.67% 和 84.70%，接近外部 RAG 的 82.57% 和 88.83%。关键是 MEGa 保持了 MMLU 分数（61.75% vs base 62.05%），而其他方法明显下降。在组合性问题（需要同时调用两条记忆）上，MEGa（49.6%/70.4%）几乎匹配 full batch training（54.4%/75.2%），远超其他 continual learning 方法。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 这是一个非常有创意的 approach——用 per-memory LoRA + gating 来解决 continual learning 是直接而优雅的。与 LoRAHub、AdapterFusion 等工作有联系但不同：MEGa 的每个 adapter 对应一条特定记忆而非一个任务。Sompolinsky 的参与带来了神经科学的深度——海马体-皮层互补记忆系统的类比不是表面的，而是在架构设计中有实质对应。
  - 🔧 **Methodology Critic**: 最大的限制是 **线性参数增长**——每条记忆一个 LoRA adapter，存储和计算成本与记忆数成线性关系。50 条记忆还好，50,000 条就不可行了。Gate 选择的准确率（85-87%）是性能瓶颈——随着记忆增多，embedding 空间越来越拥挤，confusion 会加剧。当前的 memory chunking 是静态的、预先定义的，与人类记忆的灵活编码不同。
  - 🔭 **Visionary**: MEGa 指向了一个令人兴奋的方向：LLM 作为认知架构的计算模型。如果 gated LoRA 可以看作"突触可塑性"，gate selection 看作"记忆检索"，那这个框架可以扩展为研究人类记忆机制的计算平台。未来的 rehearsal-based consolidation（sleep 期间将 LoRA 蒸馏到 base weights）直接对应海马体-皮层的记忆巩固过程，可能为认知神经科学提供新的计算假说。

- **Build-On Directions**:
  1. **Memory consolidation through rehearsal**: 实现论文提到的未来方向——通过 rehearsal 将 LoRA weights 渐进蒸馏到 base model，模拟睡眠期间的记忆巩固，解决线性参数增长问题。
  2. **Hierarchical memory organization**: 引入类似人脑的 semantic memory 组织——相关记忆共享部分 adapter 参数，形成 memory clusters，提升存储效率和组合性推理能力。
  3. **Episodic memory for AI agents**: 将 MEGa 集成到 AI agent 中，让 agent 在与用户的持续交互中积累 episodic memory，实现真正的个性化和长期关系建立。

- **Trend Signal**: 反映了 LLM 研究从 "how to build bigger models" 到 "how to make models learn continuously" 的范式转移。也反映了 neuroscience-inspired AI 的复兴——不是简单的类比，而是用计算模型严格地实现神经科学假说并在 AI 系统中验证。

## 7. Humanity's Last Exam (HLE)

- **Authors & Venue**: Long Phan, Alice Gatti, Ziwen Han, Nathaniel Li, + 1,115 co-authors from 500+ institutions. Center for AI Safety (CAIS) & Scale AI. **Nature 2026**. [Paper](https://arxiv.org/abs/2501.14249) | [Leaderboard](https://lastexam.ai)

- **Core Problem**: Benchmark 饱和——LLM 在 MMLU 等流行 benchmark 上已超过 90% 准确率，使得无法有意义地区分前沿模型或追踪推理能力的真实进展。现有 benchmark 不再挑战 state-of-the-art 系统。没有足够难的 benchmark，研究者和政策制定者无法准确评估 LLM 的真实能力和局限。

- **Key Contribution**: 论文构建了 **Humanity's Last Exam (HLE)**——2,500 道公开 + 500 道私有的多模态 benchmark 题目，处于人类专家知识的最前沿，被设计为"最后一个同类封闭式学术 benchmark"。题目由近 1,000 名学科专家从 50+ 国家 500+ 机构贡献，经过 LLM 预验证（能答对的题目被淘汰）和多轮专家审核。覆盖 100+ 学科，76% 为短答案题（精确匹配），24% 为选择题，~14% 需要多模态理解。$500,000 奖金池激励了高质量投稿。

- **Method**: 四阶段 pipeline：(1) 投稿阶段——$5,000/题奖励 top 50，$500/题奖励 next 500；(2) LLM 预验证——超 70,000 次 LLM 尝试，淘汰能被答对的题目，约 13,000 题进入人工审核；(3) 专家审核——1-3 名研究生级审稿人按标准 rubric 评审；(4) 组织者批准——最终筛选。评估严格自动化，无部分得分，答案必须精确匹配或逻辑等价。数学占 41%，生物/医学 11%，CS/AI 10%，物理 9%。

- **Results**: 2025 年 1 月首发时，最强模型 DeepSeek-R1 仅 9.4%，o1 为 9.1%，GPT-4o 仅 3.3%。所有模型 RMS 校准误差超过 80%——对错误答案给出高置信度。一年后（2026 年初），Gemini 3.1 Pro Preview 达到 45.9%，GPT-5.4 达到 41.6%，Claude Opus 4.6 Thinking Max 达到 34.4%——约 4-5 倍提升。人类领域专家在各自领域约 90% 准确率。Thinking mode 显著提升分数（34.4% vs ~19% without thinking）。多模态题目表现明显更差。独立调查（FutureHouse）发现约 30% 的化学/生物题目答案可能有误。

- **Multi-Perspective Review**:
  - 🔬 **Domain Expert**: 发表在 Nature 说明了这个 benchmark 的影响力。1,119 位作者的协作规模前所未有，确保了题目的专家级难度和学科广度。与 MMLU、GPQA、ARC 等之前的 benchmark 相比，HLE 的题目难度确实是质的飞跃。校准误差分析（模型对错误答案的过度自信）对 AI safety 有直接意义。
  - 🔧 **Methodology Critic**: 数学占 41% 的学科偏斜可能导致总分不能反映模型的全面推理能力。审稿人"不超过 5 分钟验证答案"的限制引入了质量风险——FutureHouse 发现 30% 答案可能有误就是证据。静态 benchmark 必然会被 data contamination 侵蚀。短答案的精确匹配评估可能过于严格——正确但格式不同的答案会被判错。
  - 🔭 **Visionary**: HLE 的意义超越了 benchmark 本身。一年内从 ~9% 到 ~46% 的进步速度暗示：如果这个趋势持续，LLM 可能在 2-3 年内达到专家水平。这个 benchmark 因此成为了 AI 能力进展的关键"里程碑"，对 AI policy 和 governance 的讨论有重要参考价值。但也要注意：benchmark 分数的提升不等于真正的推理能力提升——可能只是更好的 pattern matching。

- **Build-On Directions**:
  1. **Dynamic HLE**: 设计一个不断自动更新的 benchmark 机制——当模型在某类题目上达到阈值后，自动生成更难的同类题目，保持 benchmark 的挑战性。
  2. **Process-level evaluation**: 不只看最终答案对错，还评估模型的推理过程——同样答对，好的推理链和猜对的应该有不同的评分。
  3. **HLE for AI agents**: 将 HLE 扩展到需要工具使用、多步信息检索、实验设计等 agent 能力的题目，测试模型在真实科研场景中的表现。

- **Trend Signal**: 反映了 AI 社区对 "benchmark 军备竞赛" 的焦虑——每个新 benchmark 的保质期越来越短。也反映了 evaluation 本身成为一个核心研究问题——如何设计不会被 saturate 的评估方式？HLE 发表在 Nature 也反映了 AI evaluation 从纯技术问题上升为科学和社会关注的趋势。
